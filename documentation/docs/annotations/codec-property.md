# @CodecProperty

Marks a `var` property for inclusion in the codec generated by [`@GenerateCodec`](generate-codec.md). Each annotated property becomes a keyed entry in the `BuilderCodec`.

**Target:** `property`, `value parameter`
**Retention:** `SOURCE` (compile-time only)

## Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `documentation` | `String` | required | Description of the field included in the codec's documentation output |
| `required` | `Boolean` | `true` | Whether the field is required during deserialization |

## Codec key derivation

The codec key is derived from the property name with the first letter uppercased.

```
isEnabled    → "IsEnabled"
homingRange  → "HomingRange"
projectileId → "ProjectileId"
```

## Supported types

Primitive types are mapped directly to Hytale codec primitives:

| Kotlin type | Codec |
|---|---|
| `String` | `Codec.STRING` |
| `Int` | `Codec.INT` |
| `Boolean` | `Codec.BOOLEAN` |
| `Float` | `Codec.FLOAT` |
| `Double` | `Codec.DOUBLE` |
| `Long` | `Codec.LONG` |

For any other type, the processor assumes the type has a `CODEC` companion property (e.g. `HomingProjectileConfig.CODEC`).

## Usage

Fields must be mutable (`var`). The `documentation` string is required and must not be blank.

```kotlin
@GenerateCodec
class HomingProjectileComponent : Component<EntityStore> {
    @CodecProperty(documentation = "Enable/Disable Homing Feature")
    var isEnabled = false

    @CodecProperty(documentation = "Maximum distance to search for valid targets when locking on.")
    var homingRange = 0.0

    @CodecProperty(documentation = "The projectile to spawn.", required = false)
    var projectileId: String = ""

    companion object : CodecProvider<HomingProjectileComponent>, ComponentTypeProvider<HomingProjectileComponent> {
        override var componentType: ComponentType<EntityStore, HomingProjectileComponent>? = null
        override val CODEC: BuilderCodec<HomingProjectileComponent> =
            CodecBuilderHomingProjectileComponent
    }
}
```

## Validators

You can stack a validator annotation on the same property alongside `@CodecProperty`:

- [`@CodecRequiredValidator`](codec-required-validator.md) — asserts the field is non-null
- [`@CodecProjectileValidator`](codec-projectile-validator.md) — validates a projectile asset reference

```kotlin
@CodecProperty(documentation = "The projectile to spawn.")
@CodecRequiredValidator
var projectileId: String = ""
```

## Dependencies

The containing class must be annotated with [`@GenerateCodec`](generate-codec.md).
